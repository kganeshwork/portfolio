01 / FPGA & Hardware Security

XTEA Cryptography System on FPGA

25WSC354 — Electronic System Design with FPGAs

VHDL System Verilog FPGA Cryptography Vivado 2019.1

Overview

Designed and implemented a high-performance, low-power cryptography system on an FPGA platform. The system combines a hardware implementation of the eXtended Tiny Encryption Algorithm (XTEA) with a mini router, replicating a real-world secure networking environment.

Key Achievements

Part 1: Implemented XTEA in VHDL with 128-bit block and key sizes. Verified against a C reference algorithm via testbench simulation in Vivado and deployed to physical FPGA hardware.

Part 2: Built a full cryptography system in System Verilog including a mini router that arbitrates access to the XTEA encryption/decryption block across multiple IP cores.

02 / Machine Learning & Embedded Systems

ML Inference Deployment on IoT Devices

25WSC334 — Deployment of ML Inference Models in IoT Devices

ANN CNN STM32 STM32CubeIDE Embedded C IoT

Overview

Investigated the deployment of multiple machine learning inference models directly onto resource-constrained STM32 microcontrollers. The project covers ANN deployment, local vs remote inference comparison, sine wave prediction, and CNN-based face classification.

Four Parts

ANN on Embedded: Deployed an ANN on STM32; analysed how compilation flags affect inference speed and memory usage.

Local vs Remote: Compared on-device and server-side inference, exploring the impact of clock frequency on model latency.

Sine Wave & CNN: Trained and deployed a sine wave prediction model and a CNN face-classification model with hyperparameter tuning to improve accuracy.

03 / Individual Final-Year Project

ML-Based Financial Market Predictor on Embedded Systems

"Can a pocket-sized computer predict the stock market?"

CNN-LSTM Python ONNX Raspberry Pi 4B S&P 500 Deep Learning

Overview

Designed and trained a hybrid CNN-LSTM deep learning model to predict short- and long-term stock market movements using S&P 500 data. The trained model was then quantised and deployed onto a Raspberry Pi 4B, demonstrating that sophisticated financial ML can run on edge hardware.

Input features included closing price, return, exponential moving average (EMA), volatility, and RSI, normalised across 2,486 trading days with a 60/20/20 train/validation/test split.

62.3% Classification accuracy
0.014 Avg. regression error
48.3% ONNX file size reduction

Architecture & Deployment

Model: Hybrid CNN–LSTM pipeline — a convolutional layer extracts local temporal features, which are passed to an LSTM to capture long-range dependencies, enabling both directional and return predictions.

Embedded Deployment: The model was converted to ONNX format, achieving a 48.3% reduction in file size (11.50 KB → 5.94 KB), enabling inference on a Raspberry Pi 4B with confidence-level filtering that improved prediction accuracy by a further 1.33%.

Benchmark: Model outperformed a random baseline (50%) and matched or exceeded several literature benchmarks on directional accuracy.
